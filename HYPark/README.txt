AIX_main.ipynb - 코랩 버전

main.py 기존 버전과 바뀐 점

1. 비동기 처리 파이프라인 구축
기존 코드의 동기식(synchronous) 구조는 음원 분리가 진행되는 동안 서버 전체가 멈추는 
Blocking I/O 문제가 발생하고 다중 접속 시 서비스가 불가능함.

asyncio와 ThreadPoolExecutor를 활용하여 음원 분리를 별도 스레드로 분리해서
서버가 음원 분리 작업 중에도 다른 요청(업로드, 페이지 로딩 등)을 즉시 처리할 수 있도록 응답성 개선.

실질적으로 프로젝트가 로컬에서 단일 요청을 하는 방식으로 테스트 및 시연하지만 이론적으로라도 동시성을 구현할 필요가 있다
생각하기에 구현함.

2. 보안 안정성 향상

사용자 입력 파일명을 그대로 사용할 경우, Path Traversal(상위 경로 접근) 공격이나 시스템 파일 덮어쓰기 위험이 존재.
정규표현식을 활용한 secure_filename 로직을 도입하여 특수문자 및 공백을 필터링.
효과: 잠재적인 해킹 위협 차단 및 파일 시스템 I/O 에러 방지

3. 모델 로딩 최적화
기존의 요청이 들어올 때 모델을 불러오는 방식에서 로드하는 방식에서 서버 구동 시점에 
모델을 GPU 메모리에 명시적으로 로드하는 방식으로 변경
음원 분리 요청 때 로딩 지연시간 감소 및 안정적인 메모리 관리 가능
